The metadata gate, repository quality gate, four platform builds, and GitHub
release publication all completed successfully. The tag is a new immutable
`v0.1.2` tag; `v0.1.1` was not moved or reused.

## Validation

- Rust format and strict Clippy (`-D warnings`) passed.
- The complete Rust workspace test run passed 390 tests with 3 model-dependent
  tests ignored and no failures.
- Rust line coverage was 89.05%, above the 80% gate.
- Extension compile, ESLint, and Prettier checks passed.
- Fresh extension coverage was 92.01% lines and 86.99% branches, above both 80%
  gates. Unit Test MCP executed the suite but reported zero test counts; GitHub CI
  and the repository push hook are the count authorities.
- The pinned VS Code 1.93.1 Extension Host smoke passed with freshly built native
  servers: both planes connected, one session identity was shared, first ingest
  produced graph context, and ADR sync reached the coordinating state.
- The Windows VSIX contained 32 runtime entries and no source, test, coverage, or
  `node_modules` payload.

## Public runtime proof

The public Windows archive was downloaded after the release workflow completed,
verified against `SHA256SUMS`, and extracted independently of local build output.
Both binaries reported the tag build identity:

- `mindleak-mcp`: `0.1.2+677cf0d094c5`
- `lodestar-mcp`: `0.1.2+677cf0d094c5`

With the same client-minted token and base label, both returned the same opaque
session identity. The public installer then completed in a fresh workspace,
smoke-tested both servers, and generated VS Code and GitHub Copilot CLI
registrations that resolved to the same installed binaries and databases. Both
installed hosts were restarted through those generated registrations and again
returned the tag build identity and one shared session identity.

The checksum-verified public Windows VSIX installed successfully, and VS Code
reported `monk-eee.mindleak@0.1.2`.

## Scope

This record proves release engineering, packaging, installation, and local
runtime behavior. It does not claim external adoption or retention. The separate
v0.1.2 independent-developer pilot begins only after this release is available.
